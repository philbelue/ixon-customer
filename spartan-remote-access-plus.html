Key Capabilities for Spartan
The features that directly address vendor access governance, machine network security, and operational control at Spartan.
🔒
All features below are included in Remote Access Plus ($375/month) — unlimited users, unlimited machines, no per-seat licensing.
🌐
LAN Access Management
Each vendor is restricted to specific IP addresses on the machine network behind the IXrouter3. A vendor supporting Press #1 cannot reach Press #2 or any other device on the line.
Vendor A → 192.168.1.10 PLC only
Vendor B → 192.168.1.20 HMI only
Vendor C → 192.168.1.30 Camera
✅
Access Request & Approval
Vendors submit a request before connecting. A Spartan engineer receives a real-time notification and approves or denies the session. No vendor connects without Spartan authorization.
1. Vendor requests → Press #1 PLC
2. Engineer notified (email + app)
3. Engineer approves + sets window Live
4. Full session log retained
⏱️
Temporary Access
Vendor access automatically expires at the end of the approved service window. No manual revocation required. Eliminates the risk of persistent, forgotten access credentials.
Session Window → 2 hrs
Auto-Expiry → No manual revoke
Re-Access → New request required
📡
Connection Monitoring & Alerting
IXON Cloud monitors gateway connectivity in real time. If a IXrouter3 goes offline or a machine drops off the network, Spartan engineers receive an alert immediately.
IXrouter3 B → Offline alert
Channel → Email · Push · Webhook
🏷️
Custom Fields
Tag and categorize every machine in the fleet with metadata specific to Spartan's environment. Enables fast filtering, reporting, and vendor scoping.
Line → Production A / B / C

Vendor → Müller / Bühler / Roper
Plant → Grand Rapids / Ionia

LiveView & Audit Trail
See all active sessions in real time. Every session is logged — user, machine, timestamp, duration. Full access history available on demand for security audits or incident review.

Operational Workflow
How Vendor Access Works — End to End
The complete workflow from vendor request to session close. Spartan maintains control at every step.
1
Vendor Requests Access
Vendor logs into IXON portal and submits access request for specific machine
2
Spartan Engineer Notified
Real-time notification via email and IXON app. Engineer reviews request details
3
Approval + Time Window Set
Engineer approves session with defined time limit. Scope restricted to specific machine IPs
4
Vendor Connects
DirectVPN tunnel opens — scoped to approved machine only. Session visible in LiveView
5
Session Expires + Logged
Access terminates automatically. Full session log retained with user, time, and duration
