ZPI Integration Strategy
IXON and ZPI Are Additive — Not Competing
ZPI's proprietary logic is a ProMach asset — not a liability to migrate away from. The question for Chris and the technical team is simply: where does IXON's connectivity infrastructure complement ZPI's analytics capability? Three integration paths, one right answer to start.
✦ Recommended Starting Point
Path A — IXON as Connectivity Layer, ZPI for Analytics

IXON handles all machine connectivity, remote access governance, and raw PLC tag data collection. Structured data flows via MQTT or REST API to ZPI for ProMach's proprietary OEE calculations. ZPI remains the analytics brain. IXON is the secure, scalable data pipe and access layer — and adds remote access governance for the entire portfolio.

  • IXrouter3 deployed on machines across brands — IXON collects tag data
  • MQTT broker publishes structured data to ZPI's data store in real time
  • ZPI runs its proprietary calculations on clean, normalized input data
  • Results surface in both ZPI and the ProMach/IXON customer portal
  • Remote access governed entirely through IXON — no change to ZPI's analytics model
  • Non-ZPI brands immediately gain connectivity and remote access without waiting for ZPI migration
Viable Path
Path B — Full IXON Platform, ZPI Logic Migrated to SDK

IXON Cloud becomes the full stack — connectivity, data storage, and customer portal. ZPI's proprietary algorithms are ported into custom IXON App Engine modules. Eliminates on-prem dependencies and enables global SaaS scale. Higher implementation effort, but positions ProMach on a single modern architecture long-term.

  • IXON App Engine used to rebuild ZPI's core calculation modules
  • ProMach engineering migrates ZPI logic into IXON SDK (Python/JS)
  • Full customer portal delivered on MSConnect — white-labeled ProMach brand
  • ZPI on-prem decommissioned on a scheduled timeline after validation
Discuss With Chris
Path C — IXON for New Brands Now, ZPI Stays at Zarpac

Run both systems in parallel. IXON becomes the standard for the 50+ brands without ZPI today — delivering immediate remote access governance and connectivity value. ZPI continues serving Zarpac integrated lines with no migration pressure. Both systems converge on a shared data layer over time at a pace ProMach controls.

  • IXON standardized across all non-Zarpac brands immediately — fast ROI
  • ZPI continues at Zarpac with zero disruption or migration timeline
  • Convergence decision made in Phase 2 based on real adoption data
  • Lowest organizational change management burden to start
